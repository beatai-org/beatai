# AI 平台之争已经打响

![](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@d862590ea9216ae63f9cd9b4163d944ec90098a3/ai-insights/2026-06/04/images/the-ai-platform-wars-have-started/01.webp)

又来了：新一轮平台霸主之争。这次争夺的当然是 AI 竞赛的胜利。

Anthropic 本周发布 Claude Managed Agents，是迄今为止表明 AI 市场进入新阶段的最明确信号。这不只是又一次模型发布，而是 Anthropic 从卖智能转向卖执行基础设施：安全沙箱、长时运行会话、编排和治理，并把这些和 Claude Code 等现有产品串到一起。

这是平台的打法，而不是模型厂商的打法。

IT 行业向来由几家激烈竞争的大平台塑造。Wintel 对 Mac、iOS 对 Android、PlayStation 对 Xbox 对 Nintendo Switch、AWS 对 GCP 对 Azure，例子不胜枚举。

硅谷对平台的痴迷有个简单的理由：全球最值钱的公司清一色是平台公司。赢下一场平台竞赛回报极其丰厚，但代价也不低。比如 Mark Zuckerberg 砸下数百亿美元想赢下 VR 平台竞赛，至今几乎没什么拿得出手的成果，这背后是有原因的。

而眼下最重要的平台竞赛，无疑是 AI 这一场，尤其是对企业 AI 技术栈控制权的争夺。

## 从模型之战到平台之战

过去这两年，AI 市场大体上像一场模型之战：更强的推理、更强的编码、更强的多模态，更高的跑分。但这从来都只是一幅不完整的图景。

当领跑者不再只发布孤立的组件，而是开始把组件之上的整个技术栈拼起来，平台之战就开始了。当下发生的正是这件事。Anthropic 进军托管运行时是一个明确的佐证；OpenAI 的 Frontier 愿景是另一个：它明确把未来描绘成共享上下文、新人引导、反馈循环、权限，以及在真实业务系统中协作的 AI 同事。

如今这套技术栈已经清晰可见。最底层是基础模型：LLM、代码、视频、图像生成。再往上是 agent 运行时：托管执行、沙箱、工具调用、会话、浏览器或电脑访问，以及长时运行的作业。

再往上是上下文层：企业搜索、连接器、记忆、带权限感知的检索，以及公司知识。接着是构建/编排层：工作流构建器、可复用的 skill、agent 协同、评估和部署。

再往上是应用界面：编码 agent、生产力 agent，最后是作为入口的通用聊天界面。横贯这一切的，是大多数人仍然低估的一层：治理——身份、权限、追踪、审计、安全和管控。

眼下有四个重量级玩家在争夺整条技术栈：OpenAI、Anthropic、Google 和 Microsoft。当然还有许多公司提供其中的零散部分，有时甚至在某个特定环节领先。

但若以史为鉴，真正要紧的竞赛是对整条平台技术栈主导权的争夺。而这四巨头是唯一够格的角逐者。

## 四巨头各自端出了什么

![](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@d862590ea9216ae63f9cd9b4163d944ec90098a3/ai-insights/2026-06/04/images/the-ai-platform-wars-have-started/02.webp)

**Anthropic** 大概是过去几个月转身最彻底的公司。不久前，Anthropic 还只是一家有着很强前沿模型、一款高端聊天机器人，以及一款越来越强的编码产品的公司。直到推出 MCP 集成协议，Anthropic 才第一次清晰地亮出平台野心。

随后，大幅扩展的 Claude Code 成了通用 agent 平台，Claude Cowork 成了生产力工具，现在又有了 agent 执行环境。Anthropic 的产品线仍有明显缺口，比如没有图像和视频生成模型，但它在企业真正看重的地方聚焦得相当好。

**OpenAI** 则可能对终局形态有着最清晰的表述。Frontier 不是被包装成一堆工具的集合，而是一个企业级平台，用来构建、部署和管理带有共享上下文、明确权限与边界的 AI agent。

尽管它最近在编码和 agent 领域落后于 Anthropic，OpenAI 正在快速补齐技术栈。Codex 如今已是一个能做并行、长时运行任务的严肃多 agent 界面，应用范围正从编码向外扩展。ChatGPT agent 把多种运行模式和连接器整合到一起。OpenAI 的运行时方案落地得可能不如 Anthropic 清晰，但战略上的走向显而易见。OpenAI 同样想成为 AI 同事的操作系统，而且模型能力的覆盖面比 Anthropic 更广。

四个玩家里，**Google** 的技术面可能最宽。在云这一侧，Vertex 已经提供了真正的 agent 运行时环境和许多其他企业级组件。在此之上，Google 正在搭建一个不容小觑的上下文与分发层，这当然得益于其强势的 Google Workspace。它有 agent 和工作流构建器，也有一批能打（但还不算热门）的编码工具。Gemini 自然是一款强势的聊天机器人产品，得益于其模型的过硬质量，包括非常受欢迎的图像和视频生成工具。

Google 的软肋历来是产品战略显得相当散乱。AI 上也是如此：外界对 Google AI 产品质量的观感，从“惊为天人”一路滑到“他们发这玩意儿时到底在想什么？”。而且各部分之间的深度整合似乎也是个老大难。

最后，**Microsoft** 或许握着最强的企业级终局牌，尽管这并不总是那么显眼。它的优势不在模型层——这家公司把自家一些专用模型，与 OpenAI、以及最近也用上的 Anthropic 的大模型拼在一起。真正的强项在于，Microsoft 早已掌控了企业里大量真正发生工作的环境：Microsoft Office/365、Teams、GitHub 等等。Copilot 在市场上当然算不上最强的聊天机器人，但它在很多公司里依旧无处不在。

Microsoft 也在用 Foundry、Copilot Studio 和各种 Azure 产品大举搭建运行时层和构建层。只不过，要在 Microsoft 一堆产品的复杂度里摸清门道并不容易。

## 它们究竟在争什么

表层的解读是，这些厂商都想从 B2B 客户那里多分一杯羹。这没错，但不完整。它们真正争的，是成为 AI 工作默认发生的那个环境。

成为工作的默认平台可以非常赚钱。问问 Bill Gates 就知道。

在以往的平台之战里，赢家通常占住了控制力和价值创造最高的那个点。PC 时代是 CPU 加操作系统这套标准组合，它又让 Microsoft 顺势统治了生产力软件。移动时代是操作系统和应用商店。云时代是控制平面和服务广度。

我们还不知道 AI 里的这个“咽喉点”会是什么。有那么一阵，模型能力似乎将决定胜负，但如今好几家大玩家的水平已经非常接近，这个说法看来不再成立了。对 B2B 场景而言，它或许是运行时、上下文和治理的组合，并与最好的模型紧密集成。

OpenAI 和 Anthropic 已经把最新模型的训练和各自的 agent 框架紧密绑定。这清楚地表明了一种垂直整合，其动因不仅是商业考量，也来自技术性能。

最终胜出的平台，会是那个让人最容易部署出可信、互联、嵌进真实工作流的 agent 的平台。这也是我认为治理值得被当成一等平台层来对待的原因。四个玩家都在谈治理，其中 Microsoft 大概经验最多。随着时间推移，这或许会成为平台之战里最不光鲜、却最重要的那一块。

## 其他玩家呢？

当然，想拥有一个跑你 agent 的平台的，不止上面这四巨头。其他玩家也在赛道上，但相比之下明显更弱，因为它们掌握的技术栈不够多。

AWS 建起了强大的基础设施，但在模型和界面上几乎没有存在感。Salesforce 正把它的 agent 基础设施作为核心产品线的延伸来推，但对非 Salesforce 客户来说没多大吸引力。SAP、Oracle 等老牌厂商的情况也差不多。

接着自然还有一批规模更小、做得更专的玩家，它们在技术栈某一层有同类最佳的产品。

这个位置很难受。看看 Cursor 就知道了，短短几个月，它就从人人钟爱的编码工具，沦落到几乎没人再提。要和平台所有者顺手提供的、能力还说得过去的工具竞争，太难了。

## 接口会是开放还是封闭？

和早先的平台之战相比，有一点很有意思：即便平台在 UX、治理和模型层面相互竞争，AI agent 技术栈底层的管道和集成点，仍可能走向标准化和开放。

历史上，平台总想牢牢把控自己的内部运作，并精确划定第三方能在哪里施展。Apple 得批准你的应用才能上 App Store。Microsoft 会告诉你，你的 Windows 应用具体能用哪些 API。云厂商则想诱使你用它们打包好的服务，而不是用开源组件自己攒一套。这一切都是为了管控和锁定客户。

而在 AI 里，我们已经看到开放标准早早地遍地开花。MCP 让外部工具和数据得以接入，所有主流玩家都在支持它。Google 把它的 Agent2Agent 协议作为开放标准推出。在不同平台之间搬运 skill 也已经相当容易。我常在 Codex（编码能力最强）里开发 skill，然后拿到 Claude 或 OpenClaw 里跑。这出乎意料地顺。

但这能一直持续下去吗？我表示怀疑。以往平台之战里的大赢家，都是那些用开放标准抢市场份额、同时又靠自家专有方案攫取价值的玩家。偶尔也有完全开放的技术栈最终胜出（比如做网站的 LAMP），但这很少见。

## 这对创业公司意味着什么

对创业公司来说，“在前沿模型上套一层漂亮外壳”或“提供一个在幕后支撑酷炫功能的基础设施工具”那种好日子，显然要到头了。大平台的野心非常宽，而且考虑到牵涉的巨额资金，它们也必须有这样的野心。

如今许多创业公司把自己定位成所谓的“平台补全者”：它们填补大平台产品线里的空白，也就是巨头们还没拿出过硬产品的地方——工作流引擎、agent 编排器、LLM 安全、记忆平台等等。这些都是好点子，市场需求也很明确。这正是其中一些公司能飞速增长的原因。

但从历史看，这是个危险的位置。如果你不到 35 岁，大概没听过 Borland、PowerBuilder、Lotus Development、Ashton Tate 或 Sybase 这些名字。它们都是 80 年代非常成功的软件公司，填补的正是 Microsoft 和 IBM 产品线里的空白。等平台所有者把手伸进它们的地盘，它们也就基本消失了。

当然，即便身处大平台林立的世界，你也照样能做出成功的公司。比如 Snowflake 和 Databricks，尽管大型云厂商都有竞品，它们仍堪称数据仓库领域最重要的玩家。只要你真有一款同类最佳、且宽到不只是别人路线图上一个功能的产品，空间就足够大。

另一种成功策略，是在某个相关层做跨平台供应商。Oracle 是最著名的例子：它之所以能成为最重要的数据库厂商，就是因为它能跑在所有大型计算平台上。

AI 里的剧情很可能也会这么演。真正能持久的机会，要么在平台之上，要么在平台之间：垂直 agent、专有的工作流 UX、特定领域的上下文、治理与合规、专门的编排，以及跨平台的抽象层。危险的位置在中间：薄薄一层应用，或依赖某一个平台、自身又不掌握更广上下文的基础设施功能。

这正是平台之战会做的事。它们压低无差异层的价值，抬高控制点的价值。

创业公司应该好好研究平台的历史。在市场这个新阶段里你怎么给自己定位，将决定你是成是败。
