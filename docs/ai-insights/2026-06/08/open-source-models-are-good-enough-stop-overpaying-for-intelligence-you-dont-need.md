# 开源模型已经够好了，别再为你用不上的智能付高价

而且，要做好准备，以防智能成本高到无法承受。

开源模型落后于前沿模型，这一点毋庸置疑。但是，它们之间的差距究竟有多大？这种差距真的那么重要吗？

人工智能领域的讨论热衷于能力排名。本周哪个模型位居榜首？哪个实验室发布了打破先前基准记录的产品？如果你在前沿实验室工作，这些都是很有意思的问题。

对于像你我这样的其他人来说，相关的问题则有所不同：

> 哪个模型对我要做的任务来说**已经够好**，运行它**要花多少钱**？

根据 Epoch AI 对开放模型和封闭模型的分析（https://epoch.ai/blog/open-models-report），自 2023 年以来，中国开放权重模型平均落后于美国前沿模型 7 个月，差距在 4 到 14 个月之间。美国国家标准与技术研究院（NIST）人工智能标准与创新中心在其 2026 年 5 月对 DeepSeek V4-Pro 的评估中也证实了这一趋势（https://www.nist.gov/news-events/news/2026/05/caisi-evaluation-deepseek-v4-pro），并记录了在软件工程和网络安全基准测试方面存在的显著差距。

![截图来自美国国家标准与技术研究院人工智能标准中心。](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@ae8908e3bd59743a2a2b81a238180b0c8149e3ea/ai-insights/2026-06/08/images/open-source-models-are-good-enough-stop-overpaying-for-intelligence-you-dont-need/01.webp)

你知道吗？

对于你所建造的大部分东西来说……这根本不重要。

大多数生产环境中的人工智能工作负载——例如摘要、分类、实体提取、客户支持路由、文档问答和代码审查——并不需要前沿模型提供的最后 10%的功能。它们需要的是稳定、快速且足够准确的响应，并且成本能够可扩展。默认认为所有任务都应该使用前沿智能的做法，正在让企业付出远超其想象的代价，这种代价不仅体现在金钱上，还体现在他们尚未察觉的依赖关系上。

这篇文章提倡一种更审慎的方法：根据任务选择合适的模型，理解前沿溢价真正合理的适用范围，并立即开始尝试使用开源模型。了解开源软件的功能需要时间学习，明智的做法是在需要之前就开始学习。

## 本文将涵盖哪些内容？

- **开源模型落后多少？** 通过基准测试，我们可以了解前沿模型是否具有可衡量的领先优势，以及差距缩小到几乎无关紧要的程度。
- **哪些任务真正需要前沿模型？** 我将分享一个三层框架，通过推理需求对工作负载进行分类，以便您可以决定哪些前沿溢价是合理的，哪些是浪费。
- **前沿模型与开源模型相比实际成本是多少？** 需要对不同供应商的成本进行硬性比较，以及大多数团队忽略的第二个成本：供应商依赖性。
- **为什么现在是进行实验的好时机。** Publius Flavius Vegetius Renatus 在他的军事著作《De Re Militari》中写下了这段适用于 LLM 的论述。

让我们开始吧！

## 开源模型落后多少？

让我们从最明确的维度开始：在需要持续多步骤推理、复杂软件工程或对抗性安全工作的任务中，前沿模型具有真正的优势。

[NIST CAISI 的评估](https://www.nist.gov/news-events/news/2026/05/caisi-evaluation-deepseek-v4-pro)在 19 项基准测试中将美国领先的模型与 DeepSeek 模型进行了比较。在 SWE-bench Verified（一项针对真实 GitHub 问题进行自主软件工程测试）中，差距显著：美国最佳模型得分为 66.7；DeepSeek V3.1 得分为 54.8（在网络安全任务中，差距更大）。

下面的截图来自 NIST CAISI 的报告。你可以看到 GPT5.5 (xhigh) 在所有维度上都优于 DeepSeek v4 Pro (max)。我用红色标出了差距非常大的区域，用绿色标出了即使差距较大但性能仍然足以媲美 DeepSeek 的区域。

![截图来自美国国家标准与技术研究院人工智能标准中心。](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@ae8908e3bd59743a2a2b81a238180b0c8149e3ea/ai-insights/2026-06/08/images/open-source-models-are-good-enough-stop-overpaying-for-intelligence-you-dont-need/02.webp)

这些差距反映了在需要跨越数十个步骤的推理链、多个工具的协调以及从中间错误中恢复的任务上的真实能力差异。

基于 600 万次两两投票的 LM Arena 用户偏好排行榜也呈现出类似的趋势。截至 2026 年 5 月底至 6 月初，排名前 15 位的都是专有模型。

![](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@ae8908e3bd59743a2a2b81a238180b0c8149e3ea/ai-insights/2026-06/08/images/open-source-models-are-good-enough-stop-overpaying-for-intelligence-you-dont-need/03.webp)

领先的开放权重模型是 GLM-5.1、mimo、Kimi 和 DeepSeek，分别排名第 16、26、28 和 32 位。

![](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@ae8908e3bd59743a2a2b81a238180b0c8149e3ea/ai-insights/2026-06/08/images/open-source-models-are-good-enough-stop-overpaying-for-intelligence-you-dont-need/04.webp)

所以，差距确实存在。如果贵公司依赖排行榜，那么您肯定希望使用所有前沿模型。但我真心相信贵公司并非如此（另外，提示一下我们稍后会看到什么……只需查看“价格（美元/百万美元）”一栏即可😉）。

那么，我们来谈谈为什么贵公司可能不需要将这些前沿模型应用于*所有*任务。

## 哪些任务真正需要前沿模型？

在选择模型之前，最有用的做法是对任务或项目进行分类。分类依据不是领域——不是“这是编码任务还是客户支持任务”——而是推理需求。这项任务需要多少多步骤推理、错误恢复和上下文整合？

以下是一个简单的三层框架：

### 低推理需求

这项任务主要包括模式匹配、提取或重写。

例如：

- 总结一份文件
- 将电子邮件分类
- 从表单中提取结构化数据
- 翻译文本
- 根据简报生成初稿
- 处理客户支持工单。

你不需要为此使用前沿模型。就像你不需要深度神经网络来对一个小的 CSV 文件进行分类一样，一个简单的启发式模型或基于树的模型就足够了。

### 中等推理需求

这项任务需要遵循多步骤指令、处理歧义，或者生成能够向领域专家证明其合理性的输出结果。

例如：

- 审查代码的正确性（而不仅仅是风格）
- 生成一份综合多个来源信息的报告
- 从复杂的法律或技术文件中提取细微信息
- 进行连贯的多轮对话，并具备涵盖多次交流的背景知识。

开源模型在这方面仍然具有竞争力。在特殊情况下（例如不常见的输入格式、相互冲突的指令、需要领域判断的输出），差距会更加明显。只有使用自己的数据进行评估，才能了解特定任务的优劣。但总的来说，对于大多数中等推理需求的任务，开源模型很可能仍然占据优势。

### 高推理能力要求

该任务涉及自主智能体处理新颖问题、复杂的多步骤推理链（其中中间错误会累积）、对抗性或安全关键型任务，或者需要多次工具调用和恢复逻辑的智能体工作流程。

例如：

- 一个自主编码智能体正在处理一个陌生的庞大代码库
- 一个研究智能体，必须综合分析和推理多个来源的信息。
- 安全分析或渗透测试。

前沿模型在这方面具有真正的优势。基准测试结果与现有模型之间的差距很大，而且与实际任务密切相关，并在生产环境中得到验证。

### 但仅仅是原始智能就是唯一重要的维度吗？

我上面提出的三层方法有点短视，因为它*只*关注推理需求。但如果我们把成本维度也加入到智能中呢？这就是下一节要讨论的内容。

## 与开源软件相比，前沿模型实际成本是多少？

在查看具体数据之前，我想分享一个我非常喜欢并且经常使用的基准测试。它来自[artificialanalysis.ai，对 364+个模型进行了基准测试](https://artificialanalysis.ai/models)，测试指标包括智能指数、速度、延迟和价格。

![截至五月底，来自 artificialanalysis.ai 的屏幕截图。](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@ae8908e3bd59743a2a2b81a238180b0c8149e3ea/ai-insights/2026-06/08/images/open-source-models-are-good-enough-stop-overpaying-for-intelligence-you-dont-need/05.webp)

使用封闭边界模型和托管开源模型之间的权衡非常明显。顺便说一下，上面截图中的价格更接近投入成本，而不是产出成本。如果我们采用官方来源的官方投入产出 token 成本，情况会更糟。

*截至发稿时，DeepSeek V4-Pro 的价格如此之低，是因为 DeepSeek 对其进行了大幅补贴（他们声称这是永久性的），而且“一旦华为 Ascend 950 超级节点在下半年大规模推出”，价格可能会更低。*

如果将 DeepSeek V4-Pro 与 GPT-5.5 进行比较，价格差异高达 7 倍。

想想看，你什么时候会愿意花七倍的价格去购买一项服务？因为，没错，这项价格高出十五倍的服务可能更智能……但智能程度真的会高出那么多吗？

如果你不相信我，那就应该相信 Uber 的首席技术官。Uber 首席技术官 Praveen Neppalli Naga 在 2026 年 4 月直言不讳地指出：

*“我得重新开始计划，因为我原先预想的预算已经远远超出了。”*

到五月份，[Uber 的首席技术官证实，该公司在短短四个月内就用完了 2026 年的全部人工智能编码工具预算](https://fortune.com/2026/05/26/uber-coo-ai-spending-tokens-claude-code/)——大约有 5000 名工程师在使用 Claude Code。按每位工程师每月约 200 美元计算，每月总支出高达 100 万美元。

将工作量中推理能力较低的部分（文档编写、测试生成、例行重构）路由到像 DeepSeek V4-Pro 这样的模型（与 Sonnet 4.6 相比，价格大约便宜 4 倍），可以将相同的预算从 4 个月延长到一年多。

理由还不够充分？那就继续往下看。

### 还有一项费用未列在价格表中。

第二个成本是缺乏多元化。

*如果价格上涨怎么办？*

如果你只使用一家商业供应商，就会面临诸多风险。最简单的例子就是，Anthropic 的价格上涨会直接影响你的损益。你将无法降低价格，只能被动接受。

*如果发生中断怎么办？*

而这还只是最简单的问题：假设你的生产服务完全依赖于他们的 API。我可以保证，在未来的某个时候，他们的 API 肯定会宕机。当你的生产系统依赖于这唯一的生命线时，你该怎么办？

[Anthropic 的状态页面](https://status.claude.com/)仅在 2026 年 5 月就记录了五起独立事件：5 月 22 日 Opus 4.7 和 Sonnet 4.6 出现持续超过 90 分钟的严重错误；5 月 28 日计费中断；5 月 29 日 Opus 4.8 多次降级。如果该 API 是您唯一的依赖项，那么每一次事件都会给您的生产系统带来漏洞。

*如果您的 API 账号被封禁怎么办？*

如果 Anthropic 的机器人认为你的账户是恶意账户（但实际上并非如此），并决定关闭你的账户，那该怎么办？

2026 年 4 月，Anthropic 公司暂时封禁了开发者 Peter Steinberger 的账号，理由是其账号可疑。Steinberger 是开源 Claude Code 客户端 OpenClaw 的创建者，该客户端拥有数千名用户。几小时后，Anthropic 的一位工程师公开澄清，Steinberger 的账号并未违反任何政策，随后他的账号被恢复。TechCrunch 对此进行了报道。Steinberger 很幸运：事件曝光，工程师及时介入，账号迅速恢复。大多数开发者都没有这样的曝光度和运气。

顺便一提，如果你还不知道的话……Steinberger 被 OpenAI 聘用了，而且他每月都在“烧掉”价值 130 万美元的 token。这剧情简直跟《权力的游戏》一样精彩！

## 为什么现在是进行实验的最佳时机？

现在没有压力，不妨尝试使用开源模型的三个理由：

首先，**当风险为零时，学习成本很低**。对一项非关键的内部任务进行并排评估几乎不需要任何时间或金钱成本。你得到的信号是真实的，而且可能非常出人意料。即使失败，也没有任何损失。现在就用无关紧要的事情进行实验，以免将来需要用到真正重要的事情上。

第二，**差距正在缩小**。我指的不是智能差距正在缩小，因为智能差距并没有缩小。我指的是能够执行通用任务的 LLM 模型正在逐渐成为一种商品。如果回到帕累托法则，用 20%的努力就能获得 80%的成果。同样的道理也适用于 LLM 模型。只有在需要前沿模型的时候，差距才会缩小。而对于其他情况，差距确实已经缩小了很多（例如，DeepSeek V4-Pro 的性能已经相当于我们曾经惊叹的旧版 Opus 模型）。

第三，正如前文所述，**供应商集中度是一种业务风险**，而不仅仅是成本风险。例如，旧金山方面在未征求您意见的情况下做出的定价决策、因您未预料到的监管要求而导致的政策变更、以及在关键客户演示期间发生的 API 中断——这些并非罕见事件。任何长期采用单一供应商策略的团队都会遇到这些问题。对冲风险的方法并不复杂：只需将部分流量转移到不同的供应商，并逐渐熟悉其运作方式即可。

### 罗马人早已预知即将发生的事。

公元 4 世纪的 Flavius Vegetius Renatus 提出的建议至今仍然适用：

> [si vis pacem, para bellum](https://en.wikipedia.org/wiki/Si_vis_pacem,_para_bellum)*,*
>
> 想要和平，就得做好战争准备。

如果将此应用于今天（2026 年上半年），鉴于目前前沿实验室对价格的大力补贴，这意味着要为价格上涨做好准备。因为价格上涨是必然的。

未来可能出现的颠覆性变革中，受影响最大的团队是那些在变革发生前从未接触过其他替代方案的团队。而受影响最小的团队则是那些在相对平静的时期就已具备相关能力的团队，因为他们明白，选择权需要实践来积累。

我们的要求并非放弃前沿模型。正如我一直强调的，它们是市场上最优秀的模型，并且在多个领域都能创造*净*价值。我们真正需要的是构建相关的知识和基础设施，以便*有选择地*使用这些模型，并提升路由层的能力。

## 结语

新模型将会发布，排行榜也会随之改变。或许，前沿模型与开源模型之间的差距还会进一步拉大。但是，**所有这些都不会改变根本论点。**

对于任何致力于交付人工智能产品的人来说，真正重要的问题并非哪个模型绝对最佳，而是哪个模型能够*足够*完成这项任务，*足够*可靠地部署，并且*价格合理*便于扩展。对于大多数生产工作负载而言，这三个限制条件指向同一个方向：开放权重模型已经足够好用，成本差异并非微不足道。

**正确的做法是在不得不做之前就开始试验。** 即使你本周开始进行一项风险较低的单项任务实验，回答一个具体的问题：这个开放权重模型在我的数据、我的任务、我的质量标准下，究竟是如何工作的？

Vegetius 写的是军队。这个原则具有普遍性。真正能保护你的准备，就是在你不需要它的时候所做的准备。
