# SkillOpt 详解：从提示工程到技能训练

深入剖析 SkillOpt 的框架、实验结果和开源实现

技能正迅速成为构建 AI 智能体的标准方法，它将专业知识和工作流程打包成可重用的指令。Claude Skills 帮助推广了这一理念，如今大多数智能体框架都依赖于某种形式的基于技能的架构。

但构建这些系统仍然没有真正的科学方法。大多数团队依赖于编写、测试和重写指令的循环，直到获得可接受的结果。

SkillOpt 采用了不同的方法。它不把技能视为固定的文本，而是将其视为可以通过评估和反馈来改进的内容，更像是训练深度学习模型，而不是提示工程。

### 目标

本文探讨了 SkillOpt 如何将模型训练的原理应用于自然语言技能，其实验结果揭示了什么，以及该方法在哪些方面存在局限性。

到最后，你就会明白：

1. **SkillOpt 的工作原理**：为什么技能可以被视为可训练的状态，四阶段优化循环如何运作，以及哪些参数会影响结果。
2. **实验结果显示**：基准测试、评估方法、性能提升和故障模式。
3. **如何在实践中使用 SkillOpt**：设置开源库，在自己的数据集上优化技能，并比较有技能优化和无技能优化时的智能体性能。

> ***先决条件：*** *对 LLM、提示工程和机器学习训练概念有基本的了解会有帮助，但并非必需。*
>
> ***工具和库：*** [*SkillOpt*](https://github.com/microsoft/SkillOpt)*、* [*SearchQA 数据集*](https://huggingface.co/datasets/lucadiliello/searchqa)* 以及 LLM 后端（Azure OpenAI、OpenAI、Anthropic 或本地 Qwen）。*

## 1. SkillOpt 内部运作原理

SkillOpt 的核心思想是，可以使用类似于神经网络训练的原理来优化技能。

### 1.1 什么是技能？为什么人工设计的技能会停滞不前？

技能是一份简洁的指令文档，用于指导一个已冻结的智能体。它具体说明了智能体应该如何收集证据、使用工具、验证结果以及构建最终答案。模型权重保持不变，执行框架也保持不变。只有技能文本会在每次运行中发生变化。有关技能及其在智能体系统中的作用的更详细讨论，请参阅我之前的文章 [*Building Claude Skills: A New Paradigm for Interacting with LLMs*](https://aipractitioner.substack.com/p/building-claude-skills-a-new-paradigm)。

如今，大多数技能都是通过以下三种方式之一构建的，而这三种方式最终都会达到瓶颈。

- **手动设计的技能：** 无论是由人类编写、由 LLM 生成，还是通过迭代提示不断完善，它们都高度依赖于先前的直觉。
- **一次性生成的技能：** 这些技能由 LLM 一次性生成，没有实际任务结果的反馈。虽然它们通常看起来结构良好且全面，但并非基于观察到的表现。
- **自我修正技能：** 这种技能允许智能体在每次事件发生后重写自己的指令。然而，如果没有系统性的评估，有效的规则可能会被较弱的替代方案所取代，导致技能退化而非提升。

**常见的局限在于缺乏优化机制。**深度学习通过受控更新、验证和系统地利用失败信号来解决类似的挑战。SkillOpt 将相同的原则引入技能优化。它不更新隐藏参数，而是更新指令；它不依赖梯度，而是依赖于从实际执行中收集的证据。最终，它实现了可衡量、可复现和可审计的技能训练过程。

### 1.2 在一个训练步骤中会发生什么？

单次 SkillOpt 迭代遵循与任何优化循环相同的模式：观察行为、找出问题所在、提出改进方案，并测试其是否真正有效。为了更具体地说明这一过程，请考虑一个 SearchQA 示例：

> *哪部 1851 年的小说以捕鲸船* Pequod *为主角？*

标准答案为：

> Moby-Dick

迭代结束时，该技能可能获得了一条新的指令，也可能保持不变。结果完全取决于所提出的修改是否能提高对未见过示例的性能。

**#1. 推广：收集行为数据**

流程首先进行测试部署。使用当前技能，冻结的目标模型回答一系列问题，同时 SkillOpt 会记录其行为。每条轨迹包含提示、模型的响应、任何工具交互以及最终答案。然后，验证器对结果进行评分。

例如，该模型可能会回答：

> *这部小说是 Herman Melville 的* Moby-Dick*。*

尽管答案在事实上是正确的，但在完全匹配评估中得分为零，因为预期答案仅仅是 *Moby-Dick*。因此，尽管模型具备相关知识，但此次测试仍然失败了。

**#2. 反思：识别模式**

另一个独立的优化模型会审查成功和失败的案例。它不检查模型参数，而是分析行为。通过比较失败案例和成功案例，它可以区分反复出现的缺陷和已经运行良好的指令。

在这种情况下，优化器会注意到许多错误示例都包含被不必要解释包裹的正确答案，而成功示例则倾向于返回简洁明了的答案。这种对比就成为了学习信号。

反思的作用类似于反向传播：它将观察到的结果转化为改进的方向。

**#3. 编辑：提出修改建议**

一旦识别出某种模式，SkillOpt 就会将其转化为对技能的具体修改。

该系统通过三个简单的编辑操作来实现：

- 添加一句话
- 删除一个句子
- 替换句子

对于以下运行示例，优化器可能会提出：

> *请使用最短的精确跨度作答，无需额外解释。*

通常会生成多个候选修改方案。这些方案会被合并、排序，并根据修改预算进行筛选，该预算限制了技能在单次迭代中可以更改的幅度。

目前，这种修改仅仅是一种假设。

**#4. 验证：在技能中获得一席之地**

在成为技能的一部分之前，所提出的改变必须证明其价值。

SkillOpt 使用一个预留的验证集来评估候选技能，该验证集在反思过程中从未被使用过。这可以防止系统针对当前批次中的异常情况进行优化，并模拟了监督学习中验证集的作用。

按照新的指令，模型现在可以回答：

> Moby-Dick

如果验证性能有所提升，则接受修改并将其整合到技能中。如果性能保持不变或下降，则舍弃修改并保留原技能。

这种验证机制赋予了 SkillOpt 优化器严格的运行规则。只有当语句展现出可衡量的益处后，才能进入技能执行流程。

完整地遍历训练集一次即为一个周期。在批次和周期之间，推广-反思-编辑-验证的循环不断重复，通过一系列经过验证的改进逐步完善技能。

![图——四阶段循环。推广、反馈、编辑流程进入验证阶段。被接受的编辑成为新的当前技能，并用于下一轮推广。被拒绝的编辑进入缓冲区。只有当预留的验证结果有所改进时，候选编辑才能成为新的技能。被拒绝的编辑作为负面反馈返回。](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@ae8908e3bd59743a2a2b81a238180b0c8149e3ea/ai-insights/2026-06/08/images/skillopt-explained-from-prompt-engineering-to-skill-training/03.webp)

### 1.3 什么因素能保持优化循环的稳定性？

SkillOpt 通过四种机制避免不受控制的自我修改，这些机制与传统机器学习优化器中使用的安全措施类似。

**1. 有界更新：文本学习率**

在神经网络训练中，学习率控制着优化器在单次更新中权重可以移动的幅度。SkillOpt 将相同的概念应用于文本，通过**编辑预算**来实现，有时也被称为*文本学习率*。

预算限制了在一次优化步骤中可以执行的添加、删除和替换操作的次数。

默认预算为每步**四次编辑操作**。在论文的消融研究中，移除此限制导致 SearchQA 的性能从 **87.1** 下降到 **84.6**。

**2. 验证：每一次修改都必须获得认可**

反思仅仅是一种假设。在成为技能的一部分之前，修改必须证明它能提升表现。

SkillOpt 会使用一个专门预留的验证数据集来评估每个候选技能，该数据集在反思过程中从未被使用过。这可以防止优化器过度拟合那些启发修改的示例。

默认情况下，SkillOpt 使用基于精确匹配准确率的**硬性门槛**。只有当候选技能的得分高于当前技能时才会被接受。对于较小的验证集，该存储库还提供了一种**软性门槛**，它使用诸如 token 重叠等部分得分指标。

**3. 记忆：从过去的错误中学习**

与简单的自我修正循环不同，SkillOpt 会记住先前优化步骤中发生的事情。

它通过两种机制实现这一点：

- **被拒绝编辑缓冲区：** 被拒绝的候选内容不会被丢弃。相反，它们会被存储为反例，并在后续的优化过程中展示给优化器。这可以减少重复错误，并阻止之前失败的编辑方向。移除该缓冲区会将 SearchQA 的性能从 **87.1** 降低到 **85.5**。
- **慢速更新：** 单个批次的数据存在噪声。即使修改效果良好，也可能因为批次差异而导致验证失败。慢速更新机制会定期从更广泛的角度重新审视训练技能。这可以平滑噪声更新，捕捉单个批次遗漏的改进，并帮助训练从局部错误中恢复。

**4. 元技能：改进优化器本身**

SkillOpt 还维护着第二个文档，称为**元技能**。

这种区别很重要：

- **技能** → 目标模型用于解决任务的指令。
- **元技能** → 优化器模型用来改进技能的指令。

把技能想象成球员的战术手册，把元技能想象成教练的笔记。球员执行战术，而教练则记录下哪些战术能赢球、哪些错误反复出现以及哪些战术不太可能奏效。即使球员看不到笔记，它也会指导未来的决策。

**好处：**

- 随着时间的推移积累优化知识。
- 提高未来编辑的质量。
- 不会增加推理时间成本。

该论文报告称，当同时移除元技能和慢速更新机制时，性能下降幅度最大，SpreadsheetBench 从 **77.5** 降至 **55.0**。

## 2. 实验的主要发现

为了评估这个问题，作者在各种任务、模型和智能体框架中测试了 SkillOpt。

### 2.1 SkillOpt 是如何评估的？

评估涵盖六个基准，涉及不同类型的智能体行为：

![](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@ae8908e3bd59743a2a2b81a238180b0c8149e3ea/ai-insights/2026-06/08/images/skillopt-explained-from-prompt-engineering-to-skill-training/04.webp)

实验使用了七个目标模型，涵盖了从较小的 GPT 变体到前沿模型和开源 Qwen 模型。**目标模型**是待调整的智能体。一个独立的**优化器模型**负责提出技能修改建议。在本文的主要配置中，GPT-5.5 用作优化器，而目标模型则根据不同的实验而变化。

SkillOpt 与几类基准指标进行了比较：

- **无需任何技能**：该模型无需额外指令即可运行。
- **静态技能**：手动设计或一次性生成的 LLM 技能。
- **提示优化器**：诸如 TextGrad 和 GEPA 之类的方法，可以根据轨迹反馈改进提示。
- **技能演化方法**：像 Trace2Skill 和 EvoSkill 这样的方法，通过重复运行来重写技能，从智能体所做的事情以及它是否有效中学习。

### 2.2 结果如何评分？

每个基准都有其自身的成功标准，但报告方式却遵循相同的模式。

每个任务都会收到来自基准测试特定验证器的评分：

- SearchQA 使用精确匹配的准确率。
- OfficeQA 和 DocVQA 将答案与参考输出进行比较。
- LiveMathematicianBench 验证最终的数学答案。
- SpreadsheetBench 评估生成的电子表格是否满足目标规范。
- ALFWorld 衡量任务完成情况。

分数被标准化到 0 到 1 之间，然后对评估分组取平均值。最终的基准分数以百分比形式报告。

阅读论文表格时，需要注意一个细节：大多数主要结果都以**提升值**的形式呈现，而非绝对分数。**+9.6** 表示 SkillOpt 相对于未应用该技能的相同模型，准确率提高了 9.6 个百分点。

### 2.3 主要结果是什么？

主要结果很明确：**SkillOpt 始终优于所有基准。**在所有 52 个评估设置中，涵盖不同的模型、基准和执行环境（Claude code 或 codex），它要么是最佳方法，要么是表现最佳的方法之一。

![](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@ae8908e3bd59743a2a2b81a238180b0c8149e3ea/ai-insights/2026-06/08/images/skillopt-explained-from-prompt-engineering-to-skill-training/05.webp)

在 GPT-5.5 上，优化后的技能比无技能基线提高了平均准确率 **23.5 个百分点**。

从表格中可以清晰地看出一个模式。

- **程序性任务的提升最为显著**（SpreadsheetBench：**+38.9，** OfficeQA：**+39.0，** LiveMathematicianBench：**+29.3）。** 这些任务受益于明确的程序、验证步骤和工具使用规则——这些正是技能中所编码的行为类型。
- **知识密集型问答任务的改进幅度较小，但仍然有所提升。** SearchQA：**+9.6，** DocVQA：**+12.4** 当模型已经知道答案时，技能的作用就比较有限了。它可以改进格式、检索策略和验证，但无法添加模型中尚不存在的知识。

同样的趋势也出现在规模较小的模型中：具体收益有所不同，但总体结论相同：**技能对前沿模型和规模较小的模型都有帮助。**

另外两项发现对从业人员尤为重要。

- **技能迁移：** 优化后的技能与训练所用的模型或环境并非紧密耦合。论文报告称，这些技能已成功迁移到：不同规模的模型、不同的执行平台以及同一领域内的相关基准测试中。
- **技能体积小巧：** 最终生成的成品体积非常轻。典型的优化技能包含大约 **300–2000 个 token**，通常只需 **一到四次修改即可完成**。生成的指令也易于检查。SkillOpt 倾向于学习简单的操作规则，而不是去挖掘晦涩的提示技巧。

### 2.4 该方法在哪些方面存在缺陷？

这种方法很强大，但也有明显的局限性。

- **受限于目标模型。** 该技能可以改进模型的运行方式，但不能改变模型的基本认知或能力。
- **取决于优化器的质量。** 较强的优化器往往能发现更好的修改，而较弱的优化器则会生成更多无法通过验证的提议。
- **计算密集型。** 训练需要反复部署、反思和验证，因此优化成本远高于一次性编写技能。
- **对验证质量非常敏感。** 门控算法的可靠性取决于保留样本的可靠性。较小的验证集可能导致难以察觉的有效改进。
- **任务相关的收益。** 最大的改进体现在结构化、程序化的任务上，例如电子表格和办公工作流程，而开放式的问答通常收益较小。

在实践中，当目标模型已经具备底层能力、验证划分足够大以提供可靠的反馈，并且任务奖励的是一致的程序而不是广泛的世界知识时，SkillOpt 的效果最佳。

## 3. SkillOpt 的实际应用

论文中的主要结果令人印象深刻，但更实际的问题是：如果你用自己实际使用的小型模型运行这个实验，训练过的技能是否比完全没有技能更好？为了回答这个问题，我们将在单个笔记本中完整地复现 [SearchQA](https://huggingface.co/datasets/lucadiliello/searchqa) 实验的一小部分，看看这些优势在实践中是否成立。

如果您想跟着做或者自己进行实验，请随时联系我，我会分享实验记录本。

### 3.1 实验设计：三组平行模型

此设置的目标是将技能作为*唯一*变量进行隔离。目标模型、数据集和评估流程在所有运行中均保持不变。唯一改变的是注入到提示中的技能文档。

三个评估组分别是：

- **基线：** 模型完全没有获得任何技能。
- **paper_ckpt：** 使用论文作者训练并未经进一步优化评估的技能。它用作迁移测试，衡量在其他地方优化过的技能在应用于目标模型而无需额外训练时是否仍然有效。
- **self_trained：** 使用运行完整 SkillOpt 优化循环后生成的技能。这衡量的是在现有计算资源和实验设置下，训练过程本身是否能够生成有用的技能。

每次运行均使用两个指标进行评估：

- `hard`：精确匹配准确率，是本文中使用的主要 SearchQA 指标。
- `soft`：token 级 F1，当答案包含正确的跨度但不完全匹配时，给予部分分数。

同时报告这两项指标可以揭示模型何时知道答案但表达方式不佳。较高的软得分搭配较低的硬得分，可以准确地指出技能可以在哪些方面发挥作用。

### 3.2 设置：模型、优化器和数据

该实验采用非对称配置，反映了真实的部署设置：

- **目标模型：** `gpt-5-nano`：一款小巧、价格低廉的模型，能够回答所有三个评估分支中的所有问题。
- **优化器模型：** `gpt-5`：一个更强大的模型，它分析轨迹并在训练过程中提出技能调整建议。它仅在 `self_trained` 组中使用。

这遵循了论文的核心设计：更强大的优化器为较弱的目标模型训练一项技能。虽然这项技能无法添加目标模型本身不具备的知识，但优秀的优化器可以为目标模型发现更优的流程、指令和格式规则。

拆分结果包含：

- **训练：** 400 个示例用于生成滚动轨迹并优化技能。
- **验证：** 预留的验收门使用了 200 个示例。
- **测试：** 1,400 个示例保留用于最终评估，在训练期间从未见过。

标准答案仅用于评分和验证，绝不会作为模型推理过程中的输入。

### 3.3 结果

在包含 100 个项目的 SearchQA 评估切片中，两个基于技能的组都优于无技能的基线组。

![](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@ae8908e3bd59743a2a2b81a238180b0c8149e3ea/ai-insights/2026-06/08/images/skillopt-explained-from-prompt-engineering-to-skill-training/06.webp)

主要结果很简单：在模型、数据和评估器保持不变的情况下，添加 SkillOpt 训练的技能比不添加技能能提高性能。

**论文对比训练模型**

- `paper_ckpt` 组并不是一个干净的域外迁移测试，因为它是在非常类似的 SearchQA 设置上训练的。
- `self_trained` 组表现最佳。但相比 `paper_ckpt`，提升幅度较小，因此不应将此解读为自训练技能总体上更优的有力论断。

**硬/软指标的划分也很有参考价值。**

基线模型的软 F1 值已高达 0.867，这意味着该模型通常能够检索或生成正确的答案范围。较低的硬 EM 值表明，部分错误可能是格式或规范化错误，而非纯粹的知识缺失。两种技能均能提升硬 EM 值和软 F1 值，表明改进源于更清晰的提取、更严格的答案格式以及更佳的线索与答案匹配。

**技能对比结果支持这一解读。**

论文技能涵盖范围广且可重复使用，包括答案规范化、证据匹配、线索解读和常见琐事陷阱。自训练技能保留了这些基础，但为本次练习增加了更严格的规则，尤其是在 `<answer>...</answer>` 格式、答案类型、日期、数量、完形填空题、文章和数字答案方面。

**注**：这并非论文的完整复现。它仅针对一个基准测试切片和一个小型目标模型进行了小规模复现。尽管如此，它仍然回答了此设置下的主要问题：训练有素的技能优于没有技能的情况，这主要是通过改进流程而非增加知识来实现的。

## 要点总结

✓ **技能是可训练的外部状态。** SkillOpt 会在保持模型权重和执行框架不变的情况下更新技能文档。每一次改进都可以追溯到明确的、可检查的编辑操作。

✓ **该循环的行为类似于优化器。** 推广、反思、编辑和验证分别对应于前向传播、反向传播、有界更新和验证检查。只有当编辑能够提升保留部分的性能时，该编辑才会保留。

✓ **控制因素很重要。** 该论文的消融实验表明，文本学习率、被拒绝编辑缓冲区、慢速更新和元技能各自都能带来可衡量的收益。

✓ **结果显著，但取决于任务。** SkillOpt 在所有 52 个报告的评估单元中表现最佳或并列最佳，在结构化、程序性任务中进步最大，在开放式问答中进步较小。

✓ **上限依然在于模型本身。** SkillOpt 可以改进模型应用其功能的方式，但无法改变功能本身。成功取决于强大的优化器、可靠的评估以及可一致衡量的任务。
