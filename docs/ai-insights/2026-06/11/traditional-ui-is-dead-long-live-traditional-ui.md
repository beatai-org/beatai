# 为什么传统用户界面永远不会消亡：GenUI 就是用非常昂贵的流水线生产的传统用户界面。

最近一位有权势的人斩钉截铁地告诉我，传统用户界面（UI）正在消亡，用户体验（UX）也将随之被扫进历史的垃圾堆。人工智能（AI）将取代传统的用户体验和用户界面，最终形成所谓的“基因用户界面”（GenUI）。还没等其他人有机会提出有力的反驳，大家就都开始为传统用户界面摇旗呐喊了。老伙计，真舍不得你离开。认识你真好，我们一起度过了美好的时光。

但我没有和哀悼者们一起去，而是独自坐在角落里苦思冥想。

![](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@f1fc9ed0cb6f99b00a26bd6d8e3cc60928827d2f/ai-insights/2026-06/11/images/traditional-ui-is-dead-long-live-traditional-ui/01.webp)

我们被告知，如果我们继续使用传统用户界面，就等于活在过去，做着毫无意义的工作，积累产品债务。但是，生成式用户界面（GenUI）非常依赖 token，而 token 成本很高。那么，为什么要浪费大量 token，为每个使用产品的用户生成一个全新的用户界面呢？为什么不继续使用传统用户界面来处理那些需要保持速度快、价格低廉的功能，而只在用户愿意付费的功能上使用人工智能呢？

## token 错误

人工智能经济感觉就像那个老牌即兴喜剧节目《谁的台词是这样的》（Whose Line is it Anyway），喜剧演员们为了赢得一场“分数无关紧要、规则随意制定”的游戏而展开竞争。领先的公司并没有从人工智能的研发或销售中获利。而且，人工智能的成本高得离谱。但他们仍然持续投入。

![《谁的台词是这样的？》的主持人 Drew Carey](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@f1fc9ed0cb6f99b00a26bd6d8e3cc60928827d2f/ai-insights/2026-06/11/images/traditional-ui-is-dead-long-live-traditional-ui/02.webp)

看看谷歌，就在上周，他们拿着锡罐四处乞讨，问大家能不能给他们点面包，或者 [800 亿美元](https://abc.xyz/investor/news/news-details/2026/Alphabet-Announces-Proposed-80-Billion-Equity-Capital-Raise-to-Expand-AI-Infrastructure-and-Compute-2026-b0myAMewCa/default.aspx)，好喂饱我们可怜的、饥肠辘辘的机器人。因为客户为此支付的费用不够，一些大公司甚至正在改变策略，撤出人工智能领域的投资。

本周，Uber 首席运营官表示难以证明人工智能成本的合理性，这一消息迅速成为各大媒体的头条新闻。Uber 的工程师们热衷于使用人工智能工具，并在今年前四个月就用光了 2026 年的全部人工智能预算。但他们似乎并没有取得多少实际成果。

> “很难在 \[工程团队使用人工智能 token\] 和 \[开发出 25%更有用的消费者功能\] 之间划清界限……这越来越难以证明其合理性。”[Uber 首席运营官](https://youtu.be/y_mQ6xLcKyc?si=x8zRK-tFoWKPWB2l&t=1690)

这仅仅是人工智能大规模倒退的开始。星巴克在部署人工智能助手几个月后就悄悄地将其停用，原因是该助手错误地计算了咖啡店的库存。必胜客也因为人工智能导致披萨外卖速度变慢，并引发了高达 1 亿美元的诉讼，最终放弃了人工智能。多邻国在遭遇强烈的市场反弹后，也放弃了人工智能优先的策略。 [麻省理工学院报告](https://mlq.ai/media/quarterly_decks/v0.1_State_of_AI_in_Business_2025_Report.pdf)指出，“尽管企业在人工智能领域投入了 300 亿至 400 亿美元……但 95%的组织却没有获得任何回报。”

**人工智能需要从投资者主导的经济模式转向客户主导的经济模式。我们需要打造真正具有客户价值的人工智能产品，让客户愿意为此支付高价。**

## 哇，你这儿可是有不少钱呢。

我理解快速行动的必要性，但我们也需要确保我们朝着正确的方向快速前进。在为未来进行设计时，我们需要考虑哪种用户界面最适合我们的产品和用户，以及人们愿意为此支付多少费用。

以下是我们需要计算的内容的简要概述：

- **构建**和维护一个解决方案需要多少**成本**？
- **运行或维护它需要多少成本**？这包括计算成本和 token 成本。
- 您的目标客户的**最高'**[**支付意愿**](https://en.wikipedia.org/wiki/Willingness_to_pay)**'**是多少？

假设你经营一家名为 Aurora Learnealis 的在线学习公司。你的团队已经就如何在产品中使用人工智能进行了深入探讨。你们一致认为，个性化功能非常适合人工智能。

是否应该构建人工智能驱动的个性化功能？如果应该，该如何构建？为了理清思路，让我们先来看一个草图，探讨如何使用三种类型的用户界面来构建个性化功能。

**传统用户界面**：您在现有用户界面中构建个性化功能。您将用户按群体划分，并使用 Cookie 向每个群体展示专门为其精心策划的内容。构建成本并不高。但由于需要持续进行内容的精心策划，维护成本很高。**在这种情况下，您会亏损**，因为略微个性化的服务并不足以证明价格上涨的合理性（尤其是在用户可能拒绝 Cookie 的情况下）。

**传统用户界面 + AI**：您构建了一个由 AI 驱动的个性化备考功能。您使用 AI 为每位用户生成一到两次个性化内容，将其缓存并在您现有的传统用户界面中提供（以控制 token 使用量）。从零开始构建此功能成本不菲。但您的研究表明，医学生愿意为高度个性化的 MCAT 考试备考付费。因此，您有可能通过这项新功能收回成本，甚至**提高利润**。

**GenUI**：使用 GenUI 构建个性化学习体验。这种全新体验是您现有 Web 应用的一个独立模块。它能够根据用户的操作实时生成用户界面。由于您可以将其添加到现有网站，因此构建成本较低。维护成本也相当低，因为所有用户界面均由 AI 创建。但 token 使用量非常高。在这种情况下，您**会亏损**，因为即使用户愿意为个性化付费，但渲染完全由 AI 生成的体验的成本仍然更高。

这个场景究竟是虚构的还是更接近现实并不重要。更重要的是，这种前期准备工作是为了确保人工智能功能值得开发，因为你的客户最终会为此付费。

## 三个用户界面的故事

与其让 GenUI 取代一切，成功的公司会战略性地组合三种不同的 UI。产品和用户体验负责人需要引导团队决定哪种 UI 最适合他们的产品。让我们来看看如何做到这一点。

## 1. 传统用户界面

三层架构的 Web 应用程序是传统用户界面的典型代表，其后端系统负责将功能传递给预先编写好的用户界面，用户再与之交互。这里我也会把传统的原生移动应用程序归入此类。

传统用户界面并不意味着它是静态的。事实上，大多数传统用户界面都具有高度动态性、响应性和灵活性。Zoom、Spotify 和 Slack 就是很好的例子，它们在没有任何人工智能功能的情况下，依然拥有非常动态且丰富的用户界面。

![Slack 的用户界面](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@f1fc9ed0cb6f99b00a26bd6d8e3cc60928827d2f/ai-insights/2026-06/11/images/traditional-ui-is-dead-long-live-traditional-ui/03.webp)

### 评分卡：传统用户界面

**成本** (⭐⭐⭐⭐) 传统用户界面 (UI) 的设计和编码人才、实践、工具和模式的成熟和丰富，使其成为一种非常经济实惠的选择。许多公司已经拥有传统 UI 产品，这使得传统 UI 更具成本效益。如果您已经拥有一个完善的 token 系统和成熟的模式库，则无需从头开始创建任何东西。

但是，传统的用户界面更新和添加新功能可能非常昂贵。为每个新功能设计和编写全新的用户界面绝非易事，因此您需要一个高度可扩展和灵活的设计系统。

**性能** (⭐⭐⭐⭐⭐) 与我们此处列出的成本类别一样，传统用户界面 (UI) 也拥有极高的性能，这得益于我们现有的成熟且丰富的技能、实践经验、工具和模式。传统 UI 可以做得非常轻量级，使其成为瘦客户端和边缘设备的理想选择。

**可靠性和准确性** (⭐⭐⭐⭐⭐) 凭借优秀的工程团队，传统用户界面可以非常可靠且准确。传统三层架构 Web 应用的行业基准可用性为 99.9%，每月允许约 43 分钟的停机时间。对于关键的面向消费者的应用，黄金标准是 99.99%，这将年度停机时间限制在一小时以内。

关于可靠性，还有一点需要说明，这不仅仅指正常运行时间。传统的用户界面之所以能让用户感到可靠，是因为它们不会频繁变化。比如我打开 Slack，我的频道总是在同一个位置，名称也始终如一。我不需要费心去理解眼前的内容，一切都非常熟悉。而熟悉的事物总是更容易使用。

传统用户界面可以做到高度准确。但人为错误和其他因素也可能导致问题。（我不得不亲自去美国护照办公室更正我儿子的护照，因为他的名字被输错了。）大多数情况下，如果输入的数据准确，输出的数据也会准确。而且，如果传统用户界面输出的数据有误，也很容易找到解决方法。

**灵活性/可扩展性/个性化** (⭐⭐) 传统用户界面可以高度动态、响应迅速且灵活。例如，一个带有拖放卡片的数据仪表板，它响应迅速，并且在移动设备上也能完美呈现。这是一个精巧的传统用户界面，但无需消耗任何人工智能 token 即可为用户提供高性能、可定制且精准的视图。

与人工智能的个性化能力相比，传统用户界面的个性化功能非常有限。你可以按用户群体划分，并使用 cookie 为每个群体展示专门为其精心挑选的内容。构建成本并不高，但由于需要持续进行内容的人工筛选，维护成本却很高。

> **传统用户界面最适合** 那些对可预测性、性能和准确性要求极高的应用。

## 2. 传统用户界面 + 人工智能

通常是一个传统的三层网络应用程序，其中添加了一些由人工智能或机器学习算法创建的内容。

Netflix 和 Spotify 就是这种用户界面的绝佳例子：它们的界面是预先编码的，人工智能生成的内容会在这个界面中呈现给用户。Spotify 的人工智能为我生成的这个“每周发现”歌单，看起来我妈会比我更喜欢。这就是我为了下次演出循环播放 Simon and Garfunkel 的《America》而自食其果的下场。这又是另一个故事了。

![Spotify 的网页用户界面配备了人工智能生成的播放列表，《发现周刊》](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@f1fc9ed0cb6f99b00a26bd6d8e3cc60928827d2f/ai-insights/2026-06/11/images/traditional-ui-is-dead-long-live-traditional-ui/04.webp)

### 评分卡：传统用户界面 + 人工智能

**成本** (⭐⭐⭐) 传统 UI + AI 的成本介于两者之间。它既具备传统 UI 的经济性，又具备 AI 的 token 成本。如果您不使用 AI 来渲染前端，而只是用 AI 生成的内容填充传统 UI，那么您可以节省 token，同时仍然为用户提供 AI 功能。

**性能** (⭐⭐⭐) 传统用户界面加载速度快，性能卓越。多年来，前端开发人员一直在探索各种方法，让原本就流畅的传统用户界面感觉加载速度更快。将传统用户界面与人工智能结合使用，可以有效缩短人工智能所需的等待时间。您可以利用缓存数据或其他方式加载传统用户界面，在用户浏览传统用户界面的同时，人工智能元素可以有充足的时间执行更繁重的任务。

**可靠性和准确性** (⭐⭐⭐) 如上所述，传统用户界面元素可以非常可靠且准确。人工智能元素则存在准确性问题，我将在 GenAI 部分详细阐述。

**灵活性/可扩展性/个性化** (⭐⭐⭐⭐) 这是传统用户界面与人工智能的完美结合点。人工智能元素可以在不改变或重新渲染整个用户界面的情况下，带来更高的灵活性和个性化。

> **传统用户界面 + 人工智能是两全其美的选择**：既保留了传统用户界面的优点，又融合了人工智能的强大功能。这种模式非常适合那些希望系统具备可预测性基础，但又愿意为更加个性化或灵活的体验付费的系统。

## 3. GenUI

这简直太神奇了。在你最需要的时候，妈妈的拥抱就是一切。

我听说了。

> “生成式用户界面（genUI）是一种由人工智能实时动态生成的用户界面，旨在提供根据用户需求和使用场景定制的用户体验。” [NN/G](https://www.nngroup.com/articles/generative-ui/)

目前，专为最终用户（而非设计师和开发者）实时模拟用户界面而构建的推理时生成式用户界面 (GenUI) 并不常见。但它们在一些常见的 AI 工具（例如 Claude、Gemini 等）中有所应用。

![Claude 的 GenUI 与聊天回复一致](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@f1fc9ed0cb6f99b00a26bd6d8e3cc60928827d2f/ai-insights/2026-06/11/images/traditional-ui-is-dead-long-live-traditional-ui/05.webp)

### **评分卡：GenUI**

**成本** (⭐) 生成式用户界面 (GenUI) 的创建成本非常高昂，因为人工智能需要使用 token 为每个用户单独创建用户界面。如果您觉得这很奢侈，那确实如此。在我看来，这就像酒店为每位客人定制套房一样。这很棒吗？当然！如果它是免费的。但是，尽管人工智能对我们很多人来说感觉很免费，但事实并非如此。下文将对此进行更详细的阐述。

我目前见过的所有 genUI 系统实际上都不是动态生成 UI 的。它们都是通过工具调用来获取某种 UI 有效载荷。我问过 Claude 它是如何生成上面示例中的地图的，你可以看到它获取的是 places\_map\_display\_v0，这似乎是一个填充了 AI 内容的 HTML 代码块。这也是[这位博主](https://michaellivs.com/blog/reverse-engineering-claude-generative-ui/)发现的。

![Claude 解释了它是如何生成上述地图用户界面的。](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@f1fc9ed0cb6f99b00a26bd6d8e3cc60928827d2f/ai-insights/2026-06/11/images/traditional-ui-is-dead-long-live-traditional-ui/06.webp)

GenUIs 的卖点在于，它可以省去设计和构建帮助用户实现目标所需的各种不同用户界面的成本。然而，我们也面临两个隐性成本：

- **一切都是全新的，需要从零开始构建。** 各个团队正在从头开始构建 GenUI 系统。许多团队正在使用 AI 将现有的 UI 组件转换为 AI 友好的有效载荷。这样做需要大量的 AI token，因此它并不像我们最初认为的那样“免费”。
- **运行时渲染需要消耗大量 token。** 如果 AI 每次用户需要功能时都需要构建用户界面，token 消耗速度会非常快。并非所有用户界面都值得花费 token 预算从头开始构建。

**性能** (⭐) 如今的人工智能速度很慢。用户需要输入提示信息，然后等待人工智能解读并渲染输出结果。而我们现在都已经能够以光速缓慢地处理多项任务，同时还能利用人工智能进行并行处理。

在一个标签页中触发一个 AI，切换到下一个标签页触发下一个 AI，再返回第一个标签页查看，然后查看手机，因为第一个标签页的任务即将完成，如此循环往复。有时等待是值得的。但并非产品中的每个任务都值得等待 AI 渲染用户界面。

![旋转的盘子。图片来源：Unsplash](https://cdn.jsdelivr.net/gh/genesislab-io/beatai-assets@f1fc9ed0cb6f99b00a26bd6d8e3cc60928827d2f/ai-insights/2026-06/11/images/traditional-ui-is-dead-long-live-traditional-ui/07.webp)

“但是人工智能很快就会变得非常快！”你可能会这么说。没错，而且……即使是最快的人工智能，也需要相当长的时间才能像加载速度最快的传统用户界面那样快速地渲染用户界面。在这场竞赛中，传统用户界面仍然遥遥领先。

**可靠性** (⭐⭐) 人类不喜欢改变。如果每次使用产品时，用户界面都由人工智能自动生成，这在某些情况下可能非常棒，但在其他情况下则会破坏信任。用户请求引起的用户界面更改是可以接受的，但其他更改则可能令人非常不快。

我最近打开 Fitbit 应用，发现它完全变了样。我猜它改名叫 Google Health 了？总之，就连步数追踪这么简单的功能，这些变化都让我*非常*摸不着头脑。我当时不仅在找步数和区间运动时间，还愣了一下。等等，他们为什么要这么做？为什么要把所有东西都改了？步数和区间运动时间以前明明就在最上面。我的区间运动时间去哪儿了？（我其实根本不知道区间运动时间是什么，但我还是对 Fitbit 应用里的这个数字挺有感情的。）

**准确度** (⭐⭐)

> “目前生成式人工智能模型存在的问题，例如幻觉和偏见，也会在生成式用户界面中造成同样的问题。” [NN/G](https://www.nngroup.com/articles/generative-ui/)

人工智能模型并非总能出色地完成任务。它们的表现呈波动性，也就是说，它们在某些方面非常擅长，而在另一些方面则非常糟糕。随着我们开发新的 GenUI 功能，我们将观察人工智能在生成优秀用户界面方面的表现。在回答以下具体问题之前，我们还有很长的测试和评估之路要走：人工智能渲染的用户界面与用户意图的匹配程度如何？输出结果的准确性如何？

**灵活性/可扩展性/个性化** (⭐⭐⭐⭐⭐) 这正是 GenUI 的优势所在。GenUI 可以为那些需要超越传统基于用户群组的个性化的产品带来极大的灵活性和个性化。最好的例子就是我们的大型 AI 聊天应用，例如 Gemini 和 Claude。我们有一个具体独特的问题，聊天体验会为我们生成一个具体独特的答案。我们在发起聊天时就决定将 token 用于我们的提示。这非常值得。

> Gen UI 最适合用户愿意为高度个性化的体验支付额外费用，而现有 UI 又无法满足这种需求的情况。

## 朋友们，请明智地领导。

> “在这个时代蓬勃发展的公司和从业者，将是那些善于深度思考的人。” [NN/G](https://www.nngroup.com/articles/state-of-ux-2026/)

与其用 GenUI 取代一切，成功的公司会策略性地组合三种不同的 UI。产品和用户体验负责人需要引导团队决定哪种 UI 最适合他们的产品。针对不同的功能使用合适的 UI，是抓住机遇、避免损失的明智之举。
