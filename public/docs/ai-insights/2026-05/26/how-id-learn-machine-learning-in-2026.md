# 假如 2026 年从头学机器学习，我会怎么做

## 如果让我重来一次

![](./images/how-id-learn-machine-learning-in-2026/01.jpg)
*图片来源：Aron Visuals，来自 Unsplash*

我现在是 Twitch 的资深 Applied Scientist，已经做了七年的生产级 ML 系统部署。**如果让我在 2026 年的今天从零重来一次，我几乎不会按当初的任何方式去走**。

说实话，眼下流传的大部分 ML 职业建议我都会无视——很多建议来自那些把"自己当年（2019 年）入行的那条路"当成教学内容的人。那条路今天已经走不通了。我能确信这一点，是因为我已经辅导过 200 多个人入行，亲眼看着整个行业发生了多大变化。

所以接下来我会一步步告诉你，换作今天我会怎么做。如果你过去三个月一直在啃微积分课本，下面这些内容可能会让你重新审视自己的计划。

## 直觉 > 数学

但当然，我当年走过一模一样的弯路。在碰到第一个模型之前，我花了好几个月先打数学底子。把数学书从头到尾读完，手推链式法则，所有的硬骨头都啃了一遍。这套办法也确实有效……最终。

只是它比该花的时间多花了好几年，而且最怪的是，**这些严苛训练并没有真正让我对"模型在实践中为什么会这样表现"形成多好的直觉**。我能讲明白梯度下降背后的数学，但当模型在生产环境里出问题时，我多少还是在猜原因。
注意，我并不是说数学不重要。当然重要。但如果有人告诉你"必须先精通微积分才能做有用的 ML 工作"，那这个人要么在读博士，要么只是在重复别人对他说过的话。

**真正需要的数学，比大家说得要少得多**。你需要：

-   一些线性代数，让你理解矩阵相乘时到底在做什么。
-   足够的概率与统计，让贝叶斯定理和分布这些东西不再吓人。
-   微积分学到链式法则，让反向传播说得通。

差不多就是这么个清单。

剩下的东西都可以在具体问题真正逼你用到时再去补。
那么，如果不刷教材，到底该从哪里开始？

我会从一件事入手——这件事我第一次学的时候不小心跳过去了，后来不得不回头补上：**对这些算法在做什么、为什么这么做，建立真正的直觉**。

我会从 3Blue1Brown 的 [Essence of Linear Algebra](https://www.youtube.com/watch?v=fNk_zzaMoSs&list=PLZHQObOWTQDPD3MizzM2xVFitgF8hE_ab) 和 [Essence of Calculus](https://www.youtube.com/watch?v=WUvTyaaNkzM) 这两个系列开始。它把数学动画化，能让你真切看见矩阵变换对一个向量做了什么，或者一个导数在几何上长什么样。

概率与统计部分，可以看 [StatQuest](https://www.youtube.com/watch?v=pYxNSUDSFH4)。Josh Starmer 有一种本事，能用一堆可爱的小图把复杂主题拆开讲，让你真的能听懂。

关键在于：**这一步还没到掌握任何东西的时候**。只是搭一张粗糙的心智地图，为下一步做好准备。

我会把这一步限制在两到四周以内，避免掉进一个陷阱——花好几个月看视频、觉得自己在进步，其实根本没开始动手做任何东西。

## 传统机器学习基础

等数学不再让人发怵，下一步是**对传统机器学习做一次高层概览**。比如，分类和回归有什么区别？树模型、线性模型、神经网络各自在做什么？同样，现在还不追求深入理解。只要别人提到"随机森林"时，脑子里能浮现一幅大致图像就够了。[Andrew Ng 的 ML Specialization](https://imp.i384100.net/o44Kq9) 在这个阶段很扎实；如果想找免费资源，StatQuest 也有一系列很棒的视频，把主流算法都覆盖到了。

之后再花点时间学**神经网络基础**。我们的老朋友 3blue1brown 是个不错的起点；其实 StatQuest 和 [Andrew Ng 的 Deep Learning specialization](https://imp.i384100.net/bOOBax) 也都是稳妥的选择。

到了这一步，已经有了基础知识，机器学习路径上接下来有一步 ROI 极高的动作。但它需要你先掌握 Python。

## ML 用的 Python

Python 是机器学习事实上的默认语言。scikit-learn、PyTorch、TensorFlow 这类库，以及基本上你会用到的所有主流库，都建立在它之上。说真的，就算抛开 ML，早一点学 Python 也是聪明的一步：**如果最后发现自己喜欢写代码、不喜欢数学，你也已经为软件工程、数据工程以及一堆相邻职业打开了门**。

这个阶段不需要什么花哨的东西。学点数据类型、控制流、函数、文件读写这些基础就行。也要熟悉 NumPy 来做矩阵运算，熟悉 pandas 来处理表格数据——一旦开始构建模型，大多数时间都是在跟它们打交道。

大部分人到这一步会打开 scikit-learn，对模型调一下 `.fit()`，拿到结果，然后以为自己理解了发生了什么。

但如果你跟当年那个初学者的我差不多，我其实并不真正知道里面发生了什么。我后来想读教材来学得更深，多少有点用，但感觉更像是在背书里的句子，而不是真的搞懂了。

## 我最喜欢的一个学习窍门

最后真正帮我把这些东西学透的，是用 numpy 从零实现这些算法。**我会按自己理解到的程度，把步骤写出来，把基本数学写下来，然后用代码把它跑出来**。逻辑回归、k-means、决策树这些东西，在走完这个过程之后，对我来说一下子就清晰多了。

当你能看着数据流过算法的每一步时，数学就不再只是为做题而做题。

这个过程同时也是直接的面试准备——ML 编码面试经常会让你用 numpy 实现的就是这些算法，提前练就等于提前占位。

但现实是，到目前为止聊到的所有这些，光靠它们都不足以让你拿到面试机会。

**我辅导过 200 多个人完成这次转行，看到的是同一个剧本反复上演**。一个人按照他认为"该做的"全都做了——上课、学材料——结果连简历关都过不去。他会私信我：我到底哪里做错了？
紧接着，另一个理论可能学得更少的人出现，开始在同样的公司拿到回电。当你是那个学得更多的人时，看到这一幕只会更困惑。

这个差距几乎总归结于两件事。第一件——也是现在要讲的——是他们 GitHub 上有什么。第二件留到最后再说，老实讲，那一件可能更重要。但先讲 GitHub。

## 做真正的项目

很多时候，那个学得更多的人，仓库里塞满了课程练习和 Kaggle notebook——问题、数据、甚至评估指标都是别人现成递到他手上的。招聘经理看到这种东西，基本看不出这个人到底能不能胜任工作。
**招聘经理需要看到的，不是"你学过"，而是"如果今天就把你招进来，你能不能做你将要做的工作"**。

那种"别人告诉你具体造什么"的毕业设计项目，作为入门学习很棒，但要达到职业水准，就必须做职业水准的东西。**这意味着：自己识别问题、自己找并准备独特的数据、系统性地评估模型选项，然后真的把模型部署到生产环境**。

举个具体点的例子。假设你有个朋友开了一家小网店，客户流失严重——比如注册之后一个月就消失了。你可以用他的交易和互动数据建一个 churn 预测模型，弄清楚哪些特征真的能预测流失，再做一个简单的仪表盘把高风险客户标出来交给他。这就是真实世界的数据、一个会反馈"不好用"的真实用户，以及一堆需要你自己拍板的决策。

或者你喜欢健身，那也许可以做一个膳食规划 app：接收饮食限制条件，输出每周计划，外加食材开销估算。范围小没关系，但每个选择都是你自己做的，最后你手上会有一个真能演示的东西。

**一个真正能撬动求职进度的项目，关键在于展示你具备生产能力**。一个跑出漂亮结果的 Jupyter notebook 只是起点。一个用 Docker 容器化、部署在 AWS 或 GCP 上、带基础 CI/CD 流水线、用 MLflow 或 Weights & Biases 跟踪实验、并对模型表现做长期监控的项目，才会让招聘经理觉得你能在真正的职业环境里运作。

而且现在生产环境里越来越多的东西不再只是经典 ML 应用，更多是在围绕 Gen AI。

## Gen AI

如果是半年前给这个建议，我可能还会花更多时间为经典 ML 辩护，告诉你先把经典 ML 的底子打牢，再去考虑 Gen AI；但最近我对此稍微改主意了。

经典 ML 仍然重要。理解模型评估这种事，需要这些基础；如果你做的是金融或医疗这类行业，你还需要懂可解释性更强的那些模型。
我并不是叫你跳过它们，前面讲的一切都仍然适用。

但坦诚地讲，连欺诈、推荐这类领域，也越来越多地在往 Gen AI 的方向走。**2026 年的 Machine Learning Engineer，既要会经典 ML，也要会 AI engineering**。

那么具体到操作层面长什么样？**从实操角度看，最重要的两块是 RAG 和 evals**。RAG 不只是搭一条检索流水线。真正重要的是：知道什么时候 RAG 是对的选择、什么时候该 fine-tuning、什么时候只需要写一个更好的 prompt，以及怎么衡量这些方案里有没有哪一个真的奏效。

这就引到了 evals——为 Gen AI 系统搭建评估流水线，如今已经是工作里相当大的一块，而绝大多数人并不知道怎么把它做好。除了这两块，还需要理解 agent 与工具调用、面向生产系统的 prompt engineering、如何在给定成本与延迟预算下挑选合适的模型，以及像 prompt injection 这种安全基础。这些里面很多我在其他视频里讲得更深，但这些是我会优先投入的方向。

边做边用 AI 来辅助你。**ML Engineer 现在在日常工作里都在用 AI 编码助手，你必须真的擅长这件事**。这远不止"在 Cursor 里开一个聊天窗口"那么简单——还需要真正熟悉 agentic coding 的最佳实践，比如怎么高效地并行驾驭多个 agent。

但与此同时，**有一点至关重要：不要把思考外包给 ChatGPT**。有一种"流畅性幻觉"——AI 把一个完美答案递给你，你看完觉得自己懂了，但其实那一份让知识扎根所需的认知劳动，你并没有付出。卡住时让 AI 解释概念、用它来给自己出题——这都没问题；但不要让它替你写代码、自己在旁边点头如捣蒜、却根本没真懂。

## 面试准备

既然聊到写代码，**我特别希望我能告诉你"数据结构与算法面试对 ML 岗位不重要"**。对一部分公司来说，如今确实没那么重要了，因为他们开始优先做 AI-first 的面试。但对很多公司——尤其是大厂——你大概率还是会遇到 LeetCode 风格的一轮，所以学习过程中要把这件事放在心上。

回到面试。前面我说过，拿到回电与拿不到回电的人之间，有两件事拉开差距。第一件——你的作品集——已经讲完了。第二件是：你认识谁。

幸运的是，**拿到内推并不需要我们是科技圈的 nepo baby**。我们可以在线下和线上主动建立自己的网络：去参加 conference、活跃在线上社区、就那些你欣赏的人的工作给他们发一条经过深思的提问。Networking 是你能做的、对求职成功率提升杠杆最大的一件事。

今天这条路和我当年学的时候已经不一样了。它更快、更注重实战，但在一个不断变化的市场里，也有更多需要覆盖的领域。

这不会容易。**但完全可以做到。我向你保证，一旦进了这一行，所有付出都值得。**
