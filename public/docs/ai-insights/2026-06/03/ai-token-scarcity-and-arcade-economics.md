# AI token 稀缺与街机经济学

算力成本节节攀升，逼着设计师重新掂量自己的预期。

![新规则下玩着 UX 设计这场游戏。一台带设计图案的街机机柜插画。](https://cdn.jsdelivr.net/gh/beatai-org/beatai-assets@5c58ae79cd94bcca8c6b9858f61f9652fa606578/ai-insights/2026-06/03/images/ai-token-scarcity-and-arcade-economics/01.webp)
*在街机经济里，用新规则玩 UX 设计这场游戏。*

在 AI 里探索，感觉就像玩街机。"一句 prompt 就能凭空召唤出任何数字体验"，这念头太诱人，撩拨着你去突破想象的边界，一遍遍迭代，逼近某个"差不多够格、能感受到潜力"的东西。

AI 像个能扩展的协作者，随时准备用破纪录的速度把你带到"就差一点"的地方；剩下的无非是打磨和耐心，目标在望。通关任务，跳过那些磨人的关卡和道具收集。

然后，game over。一条 token 用量警告，氛围瞬间凉透。眼看再来一把就能终于通关，偏偏用量到顶了。

钱包呢？得牺牲午饭钱再投一枚硬币……趁着赢的代价还没贵得离谱。

## 经济抽象层

人们常把 AI 说成"新的"[抽象层](https://en.wikipedia.org/wiki/Abstraction_layer)，它把繁琐的技术细节藏起来，让我们专注真正要紧的事。既然能把指令翻成更易读的语言（比如 Python 或 JavaScript），何必去学机器码（一堆 0 和 1）？

AI 替我们挡掉的最常见那层抽象，其实是维持它整套基础设施所需的经济系统。

每生成一次回复，背后都跑着昂贵的 GPU、庞大的云基础设施、电力、散热系统、网络硬件等等。微软、亚马逊、Alphabet、Meta 计划在 2026 年为 AI 基础设施投入约 6300 亿到 6350 亿美元（[来源](https://www.reuters.com/commentary/breakingviews/how-big-techs-630-bln-ai-splurge-will-fall-short-2026-03-26/)）。

一条简单 prompt 的成本，我们没法完全追溯——工作日里生成的每一份访谈纪要、每一版线框图、每一个原型，背后都藏着看不见的经济外部性。放心，无论你的套餐花多少钱，真实成本都不会体现在你的订阅账单上。

麦肯锡估算，到 2030 年，全球数据中心要跟上算力需求，需要 6.7 万亿美元的资本支出（[来源](https://www.mckinsey.com/industries/technology-media-and-telecommunications/our-insights/the-cost-of-compute-a-7-trillion-dollar-race-to-scale-data-centers)）。当前全球国内生产总值（GDP）约 126 万亿美元，也就是说，光为了跟上节奏，人类就得拨出 5% 的经济产出。

把这 6.7 万亿放到相对尺度上看：同样这段时间，只需投入 930 亿美元（占 1.4%）就能消除全球饥饿（[来源](https://news.un.org/en/story/2025/11/1166397)）。所以，是在这十年结束前从饥饿中救下约 3600 万人（[来源](https://concernusa.org/news/world-hunger-facts/)），还是关掉几个 AI agent？真难选 /s。

正是这套疯狂的市场规则，看不到"喂饱相当于当前加拿大全国人口那么多人"能带来什么回报。看不到阻止全球变暖、预防下一场大流行、清理太平洋大垃圾带有什么价值。

任何投资都有机会成本：投出一美元，就放弃了把这一美元花在别处的选项。一笔好投资，回报率得高于标普 500 指数的平均水平（10%）。一群极聪明的人正把所有鸡蛋押进 AI 这只篮子——所以隔壁那座把土地、电力基础设施、先进散热都算进去后造价约 380 亿美元的数据中心（[来源](https://epoch.ai/data-insights/ai-datacenter-cost-breakdown)），下一财年理应至少带回 38 亿美元的回报。

这么一比，那笔 21 美元的 ChatGPT 订阅突然显得超划算——要么就是这些数字没能说出全貌：AI 压根算不上一笔理性的投资。

## 闪电式扩张与被补贴的 token

这标题听着像 Smashing Pumpkins 的下一张专辑——前提是 [Billy Corgan](https://en.wikipedia.org/wiki/Billy_Corgan) 的灵魂被 [Blackstone](https://en.wikipedia.org/wiki/Blackstone_Inc.) 收购了——但它其实是两套众所周知的市场支配策略的组合。

> "闪电式扩张意味着你愿意为速度牺牲效率，不等确认这份牺牲到底值不值就先冲。" —— [Reid Hoffman](https://www.strategy-business.com/author/Reid+Hoffman) 和 [Chris Yeh](https://www.strategy-business.com/author/Chris+Yeh)，[The Blitzscaling Basics](https://www.strategy-business.com/article/The-Blitzscaling-Basics)。

把增长放在第一位，投资人就来补贴庞大的算力成本，让 AI 用起来便宜，借此圈住用户、开发者和企业客户，把竞争对手挤到出局。经典案例：Uber 一直提供廉价打车，直到出租车公司倒闭，然后才掉头去收回投资。

OpenAI 从没报告过一个盈利的季度，而 Anthropic 最近（2026 年第二季度）报出 5.59 亿美元的盈利（[来源](https://www.reuters.com/business/anthropic-nears-first-quarterly-profit-agrees-pay-spacex-125-billion-monthly-2026-05-21/)）——不过这并不被指望成为常态，更何况它每月还要为 SpaceX 的 AI 算力掏 12.5 亿美元。

AI 公司把免费和打折的 token 当成长期策略，不计代价地想抢下尽可能大的市场份额。这就是为什么 Anthropic 几乎每周发一个新功能，所有人都在抢"本月最强 LLM"的头衔。闪电式扩张里没有第二、第三名的奖牌，赢家通吃。

## 充裕：亢奋与试验

在低成本 token 经济里，C 级高管被裹挟着尽可能多用 AI，鼓吹 [tokenmaxxing](https://thenewstack.io/lanai-token-tuner-tokenmaxxing/)（把 token 用量拉满）才是生产力最纯粹的表达。团队被迫在路线图里硬塞 AI 功能来推 AI 项目，哪怕这些功能粗制滥造；个人则得拉高 token 消耗，好在排行榜上显得"AI native"。

> "对你公司真正要紧的是：我的员工有没有变得疯狂沉迷 AI？而这需要让他们进入这种 tokenmaxxing 的心态。" —— Sonya Huang，红杉资本合伙人（[来源](https://www.wsj.com/cio-journal/why-some-companies-say-ai-tokenmaxxing-is-key-to-survival-e699a128?utm_source=chatgpt.com)）

而在这股充裕与压力的夹缝里，真实的用例确实被锤炼出来了：一些设计师得以探索解决方案、加速流程、搭建组织信息的新结构、重新定义工程交接，以及许多重新审视"方案如何诞生"的正当路径。

这些新工作方式，全都建立在几个假设之上：AI token 会一直随取随用，模型只会越来越好，人机增强的可能性会随着你接触这些功能的时间增长而提高。

算力成本的真实经济现实，很快就会把这层抽象撕开。许多公司的财务部门，已经在就 AI 预算开非常接地气的对话。最近 Uber 报告称，短短四个月，公司就烧光了 2026 年的整年额度（[来源](https://www.forbes.com/sites/janakirammsv/2026/05/17/uber-burns-its-2026-ai-budget-in-four-months-on-claude-code/)）。

一旦 token 的真实成本反映到账单上，下面这类话：

> "如果那个年薪 50 万美元的工程师没消耗掉至少 25 万美元的 token，我会非常警觉。" —— **Jensen Huang，英伟达 CEO**（[来源](https://www.businessinsider.com/jensen-huang-500k-engineers-250k-ai-tokens-nvidia-compute-2026-3)）

就会变成更接近这样：

> "有两样东西是无限的：宇宙和人类的愚蠢；而宇宙我还不太确定。" ―― **Albert Einstein**

## 街机经济学

四月中旬，[Claude Design](https://claude.ai/design) 正式发布。对我来说，这终于标志着出现了一款承认设计这门职业的复杂与独特之处的工具——此前的种种尝试，都在硬把 UX 工作塞进为软件开发量身定制的界面里。随便别人怎么说，[CLI](https://en.wikipedia.org/wiki/Command-line_interface) 从来没让我有过灵感。

刚一上手，我就传了一个 .fig 文件，里面装着我几年前完成的某个项目的完整设计系统。看着那些当年让开发者协调了好几个月的详尽描述，短短 15 分钟就被可视化出来，实在惊艳。尽管有些明显的短板，我还是被震到，又上传了第二个设计系统文件。

接着我发现，不到半小时，我就烧光了额度，订阅之外还多吃了 15 美元的账单。

![一个没了 token 的玩家输掉了 Boss 战，他得再投钱。](https://cdn.jsdelivr.net/gh/beatai-org/beatai-assets@5c58ae79cd94bcca8c6b9858f61f9652fa606578/ai-insights/2026-06/03/images/ai-token-scarcity-and-arcade-economics/03.webp)
*没 token？没乐子！*

和 Claude Design 这趟同行短暂又刺激，却让我看到一个灰暗的未来：创意工作的根基，也许不再主要由人的专长驱动，而是被一个封闭环境里可供消耗的算力额度多少所商品化、所增强。

换句话说，假如你是个有竞争力的 [Street Fighter II](https://en.wikipedia.org/wiki/Street_Fighter_II) 玩家，参加国际赛事得用上街机版，那预算里就必须留一块买币。可一旦因经济波动，一枚硬币的价值从 25 美分涨到比如 50 美分，就只有资源雄厚的玩家才付得起出类拔萃所需的训练时长，而那些已经在大赛上赢得盆满钵满的顶尖选手，反倒少了竞争。要是本地货币比美元更不值钱，情况只会更糟。

## 在街机经济里靠设计取胜

钱不等于创意或才华，而且很可能随着时间推移，会出现更好的策略，从每一枚币里榨出更多游玩时间——在游戏本身之上又叠了一层复杂度。

对你所用、所习以为常的资源培养一份觉察，会变得至关重要。把你迈出的每一步，都想象成一个计量经济的一部分。每条 prompt 都消耗一枚 token，驱动着同一套消耗后勤、能源、硬件、资本和政治影响力的基础设施。

很快你会意识到，并非每项 UX 活动消耗的 token 都一样多。如果你的任务需要大量上下文、又要在很长一段时间里反复推理，那它会贵到让你吃惊。

比如多个团队上传访谈逐字稿、维护研究资料库、铺开庞大的用户旅程图和动态摘要的研究综合。又或是评估完整设计系统、可访问性与分析报告、实时客户反馈的大型设计审计。

干活时，要拿到"自己到底花了多少 token"的可靠数字其实很难，你可以用 [tokenizer](https://platform.openai.com/tokenizer)（分词器）取个近似值，慢慢培养一点直觉。一般来说，如果你用英文输入，一个 token 大约相当于四分之一个单词。其他语言可能更费 token，得在此基础上再加一个百分比。

所以像下面这句、共 24 个字符的短语，算作 6 个 token。

礼貌是要付代价的。加一个简单的 please，就到了 7 个 token、31 个字符。

*Open AI tokenizer 对一句话中 token 的可视化。*

所以，文字更多并不必然意味着 token 更多；还取决于这些文字出现在哪儿。如果把任意 AI 系统看成一组输入、吞吐和输出，你会看到下面这些规律在起作用：

1.  **上下文大小——输入的边界。**单凭输入并不强大，得有一个承载逐字稿、设计系统、skill、文档和对话历史的上下文窗口。上下文提升输入质量，为后续步骤打底。很难判断多少才算"刚好够"，而求稳、把信息尽量塞满又特别诱人。
2.  **输出长度——输出的定义。**生成一份 20 页的报告显然比一份执行摘要更吃资源，所以对预期的格式、长度和信息用途说清楚、说在前头，非常重要。
3.  **对 Agentic / LLM harness 的认知——吞吐。**你对 AI harness 的选择——也就是那套直接掌控 AI 模型能力的系统、流程和接口的集合——既关键又极其微妙，所以我想在下一节多花点篇幅讲它。

### AI harness：UX 工具箱里的新工具

关于 AI harness 最好的定义，我是在 Reddit 上找到的。

> 一群没装 harness 的 LLM，就像狗公园里的一群狗。一群配上精心调校工作流的 LLM，则是一支雪橇犬队。 —— [**Sleepnotdeading**](https://www.reddit.com/user/Sleepnotdeading/)

总的来说，Gemini、Claude、ChatGPT 这类系统的网页界面，只是底下那座庞大得多的结构露出水面的尖角。水面之下还有无数其他组件，随便举几个：系统 prompt、对话管理、记忆、工具访问、文件处理、安全系统、上下文管理、用户认证、输出格式化、分析统计。

当一个用户选择 Opus 4.8，或是怀念 GPT4 的个性时，我们真正体验到的，是与那些模型交互的那层 harness。

选对 harness，等于理解效率与效能的区别。**效能**关乎达成一个清晰、明确、想要的结果。**效率**则聚焦于达成同样结果（或同样有效的结果）所需的资源量。

两家机构可能用着同一个模型，但 harness 更好的那家，通常能拿到更高的效能、更大的效率和更强的 ROI。举个例子，一家公司肯花时间去搞懂怎样安全地接上一个 [model context protocol](https://www.anthropic.com/news/model-context-protocol)（MCP），从而打通对 Figma Make 的访问。

没有 MCP，用户得手动导出画板和截图；有了 MCP，模型就能直接访问 Figma 数据。

从**效能**看，AI 更可能生成有用的设计建议，因为它接上了真正的事实源。从**效率**看，这家公司减少了手动操作、重复上下文、prompt 长度和 token 消耗。

一个好的 harness，哪怕跑在便宜的本地模型上，在简单又高度重复的任务里也可能更占上风。为了让一个前沿模型的 agent 完成某个动作，花上好几周去修它——而这动作用 make 里一个简单自动化、或干脆复制粘贴就能搞定——并不是对公司时间和算力最好的用法。

## 不必着急，回撤一点也没关系

我们都在努力适应 AI 带给职场和工作流的现实，许多讨论聚焦于它那些激动人心的能力。但也可能存在一种压力，催着人去拥抱这项技术，却没有充分掂量街机经济的潜在风险——这种经济偏爱计量供应的机器智能，胜过当下可用的人力资源。

如果你的团队已经全面"AI native"，那有必要反思一下：在没有备用方案应对潜在中断、模型更新和价格波动的情况下，完全押注 AI，等于过度信任一个很快就得把股东投资置于用户满意度之上的行业。

如果你的团队还在观望、又不知从何下手，那最重要的参数是什么让你觉得对。如果你本就在为用户和公司创造价值，那整合新技术的方法本就有扎实的根基。别丢了你的原则，让本领保持锋利——哪怕转向之后用得不那么频繁。

最有韧性的 UX 团队和个人贡献者，会是那些懂得如何把有限资源价值最大化的人：带着审视和好奇去驾驭新能力，而最重要的是，不偏离这个分寸——AI 是设计这场游戏里的一个道具，它不能变成驱动我们工作流、决策和身份认同的主引擎。

慢慢来，留出时间思考，有助于养出产品直觉和判断标准。嚼过多个选项之后，把一个想法端出来的那份滋味，里头有太多乐趣。别跳过那些关卡；一路速通冲向下一场 Boss 战自有它的价值，但真正让我们玩下去的，是对设计这场游戏的热爱。尽情享受吧！
